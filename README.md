# Marketing SQL Practice

This repository contains SQL queries designed to practice digital marketing analytics using hypothetical campaign data. The goal is to develop practical skills in analyzing advertising metrics and making data-driven decisions.

![SQL Example](https://img.icons8.com/color/48/000000/sql.png)  <!-- optional icon for style -->

## Dataset

The file `ad_campaigns.csv` contains a small, realistic dataset of ad campaigns across multiple channels, including:

- TikTok  
- Instagram  
- YouTube  
- Facebook  
- LinkedIn  

Columns included:

- `campaign_name` — Name of the campaign  
- `channel` — Advertising platform  
- `impressions` — Number of times ads were shown  
- `clicks` — Number of clicks received  
- `spend` — Amount spent on the campaign (USD)  
- `revenue` — Revenue generated from the campaign (USD)

---

## SQL Queries Included

- **CTR (Click Through Rate) by campaign**  
- **CPM (Cost per 1,000 Impressions) by campaign**  
- **ROAS (Return on Ad Spend) ranking**  
- **Channel performance summary**  

These queries are designed to provide insights into which campaigns and channels are performing best and how marketing spend can be optimized.

## Tools

- SQL (practice queries)

## Key Insights

- Retargeting campaigns tend to have higher CTR and ROAS  
- Awareness campaigns generate higher impressions but may have lower direct conversions  
- Comparing performance across channels helps identify where to allocate marketing budgets
